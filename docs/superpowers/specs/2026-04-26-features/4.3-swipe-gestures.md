# 4.3 — Swipe gestures between Survey questions

**Priority:** P3 · **Effort:** S · **Theme:** Mobile

## Problem
Touch users on phones currently tap small Next/Prev buttons. Native-feeling apps let you swipe.

## Solution
Listen for horizontal swipe gestures on the Survey's question card area. Threshold: 80px horizontal travel, <500ms duration, mostly horizontal (vertical < 50px). Direction depends on locale:
- LTR: swipe left → Next, swipe right → Prev
- RTL: swipe right → Next, swipe left → Prev

Use plain pointerdown/pointermove/pointerup; no library.

Disable when an input/textarea/select inside the question is focused (e.g., text questions) — those need normal text-selection gestures.

## Acceptance criteria
- [ ] Swipe across the question area triggers Next/Prev
- [ ] Direction respects RTL/LTR
- [ ] Doesn't trigger when interacting with text inputs
- [ ] Doesn't conflict with vertical scrolling
- [ ] Works on both touch and pen input
