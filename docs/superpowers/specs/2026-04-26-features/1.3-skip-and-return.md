# 1.3 — Skip + come back to questions

**Priority:** P1 · **Effort:** M · **Theme:** Survey completion

## Problem
Some questions need cross-team input ("how many engineers do you have?", "what's the hourly cost?"). Forcing a linear answer order means a respondent who doesn't know one answer abandons the whole session.

## Solution
Add an explicit "Skip" button next to Next on each question. Skipped questions are tracked in a `skipped: Set<questionId>` per respondent. On the last question, instead of going straight to Analyze, show a "Review skipped (N)" panel that lets the user navigate back through unanswered questions before finishing. If they're satisfied with skipping, an "Analyze anyway" button proceeds.

Skipping is distinct from "no answer" — a skipped question explicitly says "I chose not to answer this" so we can render it differently in Results (gray vs missing).

## Acceptance criteria
- [ ] Skip button on every question
- [ ] Skipped state persisted to localStorage
- [ ] On the last question, a review panel lists skipped questions with click-to-jump
- [ ] User can change a skipped answer, which removes it from the skipped set
- [ ] Analyze button enabled regardless of skip state, with a count summary
- [ ] Skipped questions show a distinct treatment in Results (faint, "skipped" tag)
