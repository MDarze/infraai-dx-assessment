# 7.1 — AI-generated executive summary

**Priority:** P1 · **Effort:** M · **Theme:** AI features

## Problem
The Results page shows structured data (scores, DNA, recommendations) but no synthesized narrative. A CFO or board member skims the report; they want a 2-paragraph summary in their voice. Writing this by hand for every assessment doesn't scale; not having one undersells the deliverable.

## Solution
After sync completes, fire a backend call to Claude API (`claude-sonnet-4-6` is fast enough; `haiku` if cost is critical). Pass the assessment payload (scores + DNA + key answers) plus a system prompt describing the desired tone (executive, action-oriented, ~150 words, AR + EN versions).

Cache the summary on the assessment record (don't re-generate on every page load). Include in PDF export.

This is also a perfect product self-demonstration — "we built this with AI; here's AI in your report."

## Acceptance criteria
- [ ] Summary auto-generated on first sync; cached on assessment record
- [ ] Available in both AR and EN
- [ ] Renders in a dedicated section on Results, near the top
- [ ] Included in PDF export
- [ ] Failure to generate doesn't block the rest of the report
- [ ] Prompt caching enabled (system prompt is large and reusable)

## Files
- Backend: Claude API integration, prompt template, cache field on assessment
- `src/pages/Results.tsx` — render summary section
- `src/api.ts` — fetch summary endpoint

## Out of scope
- User edits to the AI summary (later, in Notes 6.6)
- Streaming the summary as it generates (just show it when ready)
- Voice generation of the summary
