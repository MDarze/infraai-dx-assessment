# 7.5 — Conversational survey mode

**Priority:** P3 · **Effort:** L · **Theme:** AI features

## Problem
Form surveys feel transactional. Some non-technical respondents respond better to conversation. A chat interface where an LLM asks the questions in natural language could lower cognitive load.

## Solution
A toggle on the Survey: "Form" (default) / "Chat". In chat mode, an LLM (Claude haiku for speed) asks each question in natural Arabic or English, parses the user's free-text response, maps it to the structured option (with confirmation), and proceeds.

The structured `answers` data is the same on the backend — just a different input modality.

This is a big swing. Risks: parsing failures, hallucinated questions, longer interactions. Worth piloting on a subset of users first.

## Acceptance criteria
- [ ] Toggle visible on Start
- [ ] Chat mode covers all 25 questions correctly
- [ ] Final answers populate the same structure as form mode
- [ ] User can switch back to form mid-flow without losing data
- [ ] Clear "I'm not sure" / "skip" handling

## Files
- Backend: chat endpoint with conversation state
- Frontend: new ChatSurvey.tsx page
- Heavy use of i18n for the LLM system prompt

## Out of scope
- Voice chat (separate, after 7.3 lands)
- Continuing a chat across sessions
