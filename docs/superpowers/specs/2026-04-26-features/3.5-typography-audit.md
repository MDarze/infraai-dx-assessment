# 3.5 — Typographic hierarchy audit

**Priority:** P2 · **Effort:** S · **Theme:** Branding

## Problem
The design spec defines a clean type scale (`12 / 14 / 16 / 18 / 22 / 28 / 36`) but components use inconsistent sizes — Survey question text uses `text-lg` (18) while Results section headers use `text-xl` (22). No clear hierarchy makes pages feel rhythm-less.

## Solution
Audit every `text-*` class in `src/`. Establish a strict pattern:
- `text-3xl` — Page H1 (Results report title)
- `text-2xl` — Section H2 (top-level sections on a page)
- `text-xl` — Card H3 (sub-section / question text)
- `text-lg` — Important label / large body
- `text-base` — Body (default)
- `text-sm` — Secondary
- `text-xs` — Meta, caption

Walk every page, fix mismatches. Verify font weight pairings: `font-semibold` for h1/h2, `font-medium` for h3/labels, default for body.

## Acceptance criteria
- [ ] Visual diff: pages look more rhythmic side-by-side
- [ ] No `text-lg`-then-`text-xl` headline jumps mid-page
- [ ] Responsive variants (sm:text-2xl etc) only where structurally needed
- [ ] Pages compared at desktop/tablet/phone for consistency

## Files
- `src/pages/*` — class adjustments
- `src/ui/PageHeader.tsx`, `src/ui/Card.tsx`, `src/ui/Stat.tsx` — verify the primitives use the right scale

## Out of scope
- Font feature settings, kerning tweaks (later)
- Per-locale type scale differences (Arabic and Latin both use the same scale here)
