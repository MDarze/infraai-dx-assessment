# 2.3 — Shareable read-only results link

**Priority:** P2 · **Effort:** M · **Theme:** Results value

## Problem
The assessor wants to forward results to a client's colleague (CFO, board), but the current Results page is local to the assessor's session. Re-running the assessment isn't an option. Email-with-PDF is fine but doesn't update if scores change.

## Solution
Backend endpoint `POST /api/assessment/{id}/share` returns a signed, time-limited URL like `/share/{token}`. Frontend has a "Share" button on Results that calls this and copies the URL to clipboard.

A new public route in the frontend (`/share/{token}`) fetches the assessment by token from a public read endpoint (`GET /api/assessment/by-token/{token}`) and renders a stripped-down Results view (no edit, no Export, no Settings).

Tokens expire after 30 days by default; revocable from the backoffice.

## Acceptance criteria
- [ ] Share button on Results creates a token and copies URL to clipboard
- [ ] Anyone with the URL sees a read-only Results page
- [ ] Token expires after 30 days
- [ ] No PII beyond what's already in the report leaks
- [ ] Backoffice can revoke a token

## Files
- Backend: token table, share/revoke endpoints, public read endpoint
- `src/App.tsx` — add `/share/{token}` route (currently no router; need to add one or use a hash route)
- `src/pages/SharedResults.tsx` (new) — read-only variant of Results
- `src/api.ts` — `createShareLink`, `fetchByToken`

## Out of scope
- Permissioned sharing (just unguessable URLs, like Google Docs share-anyone)
- Comments / annotations on shared link
