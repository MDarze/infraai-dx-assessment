# 3.8 — Print stylesheet for in-page printing

**Priority:** P2 · **Effort:** S · **Theme:** Branding

## Problem
The current PDF export opens a new window with hand-rolled HTML and triggers print. It's fragile (popup blockers), looks different from what the user sees, and doesn't let them just hit Cmd+P to print the live page.

## Solution
Add a `@media print` block in `src/styles.css` that styles the live Results page for paper:
- Hide AppHeader, Footer, action buttons
- Force light theme regardless of user preference
- Avoid background-image patterns
- Set page margins (0.5in)
- Use `break-inside: avoid` on Card sections
- Force columns to single column at print width

Once this exists, the existing `exportPdf` window-print hack becomes optional — eventually replace with proper PDF (2.1).

## Acceptance criteria
- [ ] Cmd+P / Ctrl+P from Results produces a clean printout
- [ ] No header/footer/buttons in printed version
- [ ] Light theme used regardless of UI theme
- [ ] Cards don't break across pages mid-row
- [ ] Works in both AR (RTL) and EN (LTR)

## Files
- `src/styles.css` — `@media print` block
- Optionally remove or simplify `buildPrintableHtml` in Results.tsx

## Out of scope
- PDF generation pipeline (covered by 2.1)
- Custom page headers/footers (browser default OK for v1)
