# 2.6 — Interactive ROI playground

**Priority:** P1 · **Effort:** S–M · **Theme:** Results value

## Problem
The ROI section shows a single estimate based on settings buried in a separate Settings page. In sales conversations the client wants to ask "what if we had 10 engineers instead of 5?" and see the number change. Currently they can't.

## Solution
On the Results ROI section, render the four input variables (`engineersCount`, `workingDaysPerWeek`, `timeSavingRate`, `avgHourCostSar`) as inline sliders. Above each slider show its current value; below, the recomputed weeklyHoursSaved + weeklyCostSavedSar update live (no save needed for the playground; original state is preserved unless user explicitly clicks "Use these as defaults").

The math is already in `analysis.ts`'s `roiEstimate`; expose it as a pure function `computeRoi(inputs)` we can call client-side on slider change.

## Acceptance criteria
- [ ] Four sliders visible in the ROI section
- [ ] Numbers update live as sliders move
- [ ] Original assessment values preserved unless explicitly overwritten
- [ ] "Reset to defaults" button restores assessment values
- [ ] Works on touch devices (sliders are accessible / tappable)

## Files
- `src/analysis.ts` — extract `computeRoi(args)` as exported pure function
- `src/pages/Results.tsx` — slider UI
- `src/ui/Slider.tsx` (new) — branded slider primitive

## Out of scope
- Saving slider values back to the persisted assessment (separate "Use these as defaults" later)
- Comparison view (slider vs assessment values)
