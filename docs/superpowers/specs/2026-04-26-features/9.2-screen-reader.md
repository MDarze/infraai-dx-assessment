# 9.2 — Screen-reader accessibility pass

**Priority:** P2 · **Effort:** S · **Theme:** Accessibility

## Problem
Visual cues (color-coded sync status, icon-only toggles, score numbers) don't convey to screen readers without explicit ARIA. Assistive-tech users get an incomplete picture.

## Solution
Page-by-page audit:
- ThemeToggle and LocaleToggle: `aria-label` describing current state
- Sync status pill on Results: `aria-live="polite"` so changes are announced
- Score number on Results: ensure full value read out (e.g., "Overall score 62 out of 100")
- Question card: question text wrapped in `<h2>` so SR users can navigate by heading
- Stat tiles: explicitly group label + value with `aria-label` if visual order is wrong
- Empty-state messages: `role="status"` so they're announced when content loads

Test with NVDA (Windows), VoiceOver (Mac), TalkBack (Android), and at minimum the screen reader simulator in Chrome devtools.

## Acceptance criteria
- [ ] Per-page screen-reader walkthrough documented
- [ ] All toggles announce current state
- [ ] Sync status changes auto-announced
- [ ] Score and stats fully read with units
- [ ] Empty states and error states announced

## Files
- Various components — add aria-* attrs
- Test results documented

## Out of scope
- Custom screen-reader-only "tour" of the app
- Voice-driven navigation
