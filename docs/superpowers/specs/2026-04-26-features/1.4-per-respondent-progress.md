# 1.4 — Per-respondent progress in role pills

**Priority:** P2 · **Effort:** S · **Theme:** Survey completion

## Problem
With 2–4 respondents, you can't tell at a glance who's behind. The active respondent has all attention; the inactive ones could be 0% or 99% complete.

## Solution
On each role pill in the Survey's role-section card, append a small fraction `12/18` showing answered/total for that respondent's visible questions (questions where `q.role === r.role || q.role === "All"`). Use the `mono` class for the digits. For the active respondent, also show a thin progress underline on the pill.

## Acceptance criteria
- [ ] Each role pill shows `<answered>/<total>` for that respondent's question set
- [ ] Numbers update live as answers change
- [ ] Active respondent has a visual differentiator (underline or stronger background) — already does
- [ ] Numbers in JetBrains Mono, locale-independent (Western digits)

## Files
- `src/pages/Survey.tsx` — compute per-respondent counts
- `src/pageHelpers.tsx` — `countAnswers` already exists; extend or reuse

## Out of scope
- Aggregate "all respondents" progress bar (different feature)
