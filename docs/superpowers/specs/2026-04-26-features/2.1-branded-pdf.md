# 2.1 — Branded PDF report

**Priority:** P1 · **Effort:** M–L · **Theme:** Results value

## Problem
The current PDF export uses `window.open` + `window.print` with inline HTML (`buildPrintableHtml` in `src/pages/Results.tsx`). It works but looks like a printout, not a $$ deliverable. No logo, no cover page, no page-aware layout, browser headers/footers leak through.

## Solution
Replace the print-window hack with a real PDF pipeline. Two viable paths:

**A) Client-side with react-pdf or pdfme** — render PDF in the browser using a declarative component tree. Pros: zero backend cost, instant download. Cons: less control over precise layout, font embedding for Arabic can be fiddly.

**B) Backend-rendered with Puppeteer / pdfkit** — frontend POSTs the report payload to `/api/report/pdf`; backend renders and streams back. Pros: pixel-perfect, server-rendered HTML/CSS. Cons: backend dependency, Arabic font hosting.

Recommend **B** for the production quality bar. Use a proper template:
- Cover page: logo, "InfraAI Field Assessment Report", client name, date, assessor
- Executive summary: hero score, DNA, top 3 quick wins
- Per-axis breakdown
- Detailed quick wins / AI opps (one per page)
- ROI section with assumptions
- Appendix: full Q&A by respondent

## Acceptance criteria
- [ ] PDF generation triggered from the same Export PDF button on Results
- [ ] Cover page with logo + client + date
- [ ] Embeds IBM Plex Sans Arabic correctly
- [ ] Page numbers + footer with assessment ID
- [ ] Tables don't break across pages mid-row
- [ ] Both AR and EN versions render correctly (separate PDFs or one bilingual document; decide before building)

## Files
- Backend repo: new `/api/report/pdf` endpoint
- `src/pages/Results.tsx` — replace `exportPdf` to fetch from backend and trigger download
- New: report HTML template living in backend repo
