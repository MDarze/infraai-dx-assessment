# 8.2 — HubSpot / Pipedrive contact sync

**Priority:** P2 · **Effort:** M · **Theme:** Integrations

## Problem
Every completed assessment has lead-quality data: contact info, company size, industry, score. That should be in your sales CRM, not just in the backend DB.

## Solution
Once 8.1 (webhook) lands, this can be done via Zapier/Make with no code. But for users who want a direct integration:

Backend supports a HubSpot or Pipedrive API key (env-configurable). On assessment complete, also POST to the CRM:
- Create/update Contact (by email)
- Create/update Company (by name)
- Add custom properties: `infraai_score`, `infraai_assessment_id`, `infraai_dna_*`
- Create a Deal in the relevant pipeline

Decide which CRM to support first based on user demand (likely HubSpot for SaaS-style, Pipedrive for sales-heavy).

## Acceptance criteria
- [ ] Configured via env vars only (no in-app setup UI for v1)
- [ ] Idempotent (re-running doesn't create duplicate contacts)
- [ ] Failure logged, doesn't block submission
- [ ] Custom properties created on first run if missing

## Files
- Backend only

## Out of scope
- In-app CRM connector UI
- Bidirectional sync (CRM → backend)
- Multiple CRM integrations simultaneously (pick one)
