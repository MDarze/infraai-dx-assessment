# 7.3 — Voice-to-text for note fields

**Priority:** P2 · **Effort:** S · **Theme:** AI features

## Problem
Field engineers in PPE on construction sites can't easily type long notes on phones. Voice input is the obvious affordance, but the platform default keyboard's mic button is awkward and not always available.

## Solution
A microphone icon button next to every text/textarea field in the Survey. Clicking starts Web Speech API recognition (browser-native, no API cost). Speech transcribed live into the field. Supports both `ar-SA` and `en-US` recognition based on current locale.

Browser support: Chrome / Edge solid; Safari iOS supported but require user permission. Show a fallback message on unsupported browsers.

## Acceptance criteria
- [ ] Mic button on text/textarea fields in Survey (text questions, future note fields)
- [ ] Live transcription into the field as user speaks
- [ ] Locale-appropriate recognition (ar-SA / en-US)
- [ ] Mic icon visually shows active recording state
- [ ] Graceful fallback message on unsupported browsers
- [ ] Permission denial handled with clear copy

## Files
- `src/ui/MicButton.tsx` (new)
- `src/pageHelpers.tsx` QuestionRenderer — integrate mic button
- `src/i18n/{ar,en}.ts` — mic state strings

## Out of scope
- Server-side transcription (Whisper) — just Web Speech for v1
- Voice command navigation
