# 5.4 — Case studies / social proof

**Priority:** P3 · **Effort:** S · **Theme:** Trust & conversion

## Problem
First-time visitors have no reference for whether others trust this assessment. SaaS standard pattern is "X firms assessed" or testimonial pull-quotes.

## Solution
Once you have 5+ completed assessments, add a small social-proof block on the Start page hero:
- "Trusted by N firms across KSA construction"
- Optional 1–2 anonymized testimonial quotes ("This assessment helped us identify a 30% saving on reporting time" — contractor name redacted unless permission)

Number can be backend-driven (`GET /api/stats/total-assessments`) so it auto-increments.

Skip until you have testimonials worth quoting; placeholder testimonials are worse than none.

## Acceptance criteria
- [ ] "N firms assessed" auto-updates from backend
- [ ] Testimonials curated, real, with redaction policy
- [ ] Skipped entirely if N < 5 or no testimonials yet

## Files
- `src/pages/Start.tsx` — social-proof block
- Backend stats endpoint
- Testimonial content stored where? (i18n or separate JSON)

## Out of scope
- Logos wall (need explicit permission per logo)
- Video testimonials
