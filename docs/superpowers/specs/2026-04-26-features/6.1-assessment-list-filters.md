# 6.1 — Assessment list view with filters (backoffice)

**Priority:** P2 · **Effort:** S–M · **Theme:** Backoffice

## Problem
Backoffice already has an assessment list (per user's mention). With 30+ assessments, scrolling is impractical. Filters / search are needed.

## Solution
Add filter controls above the list:
- Date range (created between)
- Score band (0–40 / 41–70 / 71–100)
- Company size (lt50 / 50to200 / gt200)
- City (text search)
- Sort: newest / score desc / score asc / client name

Server-side filtering is cleanest (`GET /api/assessments?score_min=...`). Pagination: 20 per page.

## Acceptance criteria
- [ ] Filter controls above list
- [ ] Server-side filtering
- [ ] URL query params reflect filter state (shareable links)
- [ ] Pagination
- [ ] Empty-state copy when filters return zero

## Files
- Backend: extend assessments endpoint to accept query params
- Backoffice frontend (separate repo)

## Out of scope
- Saved filter presets
- Bulk actions
