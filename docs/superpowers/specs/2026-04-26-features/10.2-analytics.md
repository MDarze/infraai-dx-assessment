# 10.2 — Funnel analytics (Plausible / PostHog)

**Priority:** P1 · **Effort:** S · **Theme:** Operational

## Problem
We don't know:
- How many people start an assessment vs complete it
- Where drop-off happens (which question kills engagement)
- Completion time
- Mobile vs desktop ratio
- Which export action gets used (PDF vs JSON vs nothing)

Without this, all UX decisions are guesses.

## Solution
Add a privacy-respecting analytics SDK. Recommend:
- **Plausible** (paid, ~$9/mo, GDPR-friendly, no cookies, simple) — best fit for a small B2B tool
- **PostHog** (free tier, more powerful, can self-host) — better if you want session recordings later

Track events:
- `assessment_started` (Start → Survey transition)
- `question_answered` (with questionId, time-to-answer)
- `assessment_completed` (Analyze click)
- `export_pdf_clicked`, `export_json_clicked`
- `share_link_created` (if 2.3 lands)
- Page views as default

No PII. Event payload is anonymous behavior, not identifiable user data.

## Acceptance criteria
- [ ] Analytics SDK loaded in production only
- [ ] Funnel events fire correctly
- [ ] No cookies / no consent banner needed (Plausible) OR consent banner if required (PostHog with cookies)
- [ ] Dashboard shows funnel from Start → Complete
- [ ] Bot traffic excluded

## Files
- Add SDK dep
- `src/main.tsx` — init
- Various places — event firing helpers

## Out of scope
- Heatmaps (PostHog has them; defer)
- A/B testing (defer)
