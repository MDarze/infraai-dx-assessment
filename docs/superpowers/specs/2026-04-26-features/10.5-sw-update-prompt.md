# 10.5 — Service-worker update prompt

**Priority:** P1 · **Effort:** S · **Theme:** Operational

## Problem
The PWA service worker auto-updates silently. But "auto-updates" only takes effect after the user closes all tabs and reopens. Users frequently get stuck on old versions for days, missing fixes — exactly the problem we hit during the locale-toggle bug ("hard refresh to bust the SW cache").

## Solution
Use vite-plugin-pwa's `useRegisterSW` hook to detect when a new service worker is waiting. When detected, show a small toast/banner:

> A new version is available. **Refresh** to update.

Refresh button calls `updateSW(true)` which skips waiting and reloads. Dismiss button hides until next session.

This eliminates the "tell users to hard refresh" pattern.

## Acceptance criteria
- [ ] Banner appears within 30s of a new SW being available
- [ ] Refresh button reloads the app on the new version
- [ ] Dismiss persists for the session only
- [ ] Banner doesn't appear on first load (only on update)
- [ ] Bilingual

## Files
- `src/main.tsx` or new `src/ui/SwUpdateBanner.tsx`
- `vite.config.ts` — ensure `registerType: 'prompt'` (currently 'autoUpdate')

## Out of scope
- Forced reload after N hours (just prompt)
- Update notes shown in the banner (just generic prompt)
