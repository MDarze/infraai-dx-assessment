# 4.2 — Tablet layout for Survey (split-pane below desktop breakpoint)

**Priority:** P2 · **Effort:** M · **Theme:** Mobile

## Problem
Field engineers will use this on iPad / Android tablets in the truck. The current layout collapses to single-column at <1024px, which wastes the tablet's horizontal space. A split-pane (rail + content) makes sense from ~768px up.

## Solution
Drop the desktop split-pane breakpoint from `lg:` (1024px) to `md:` (768px). The respondent rail (1.4) becomes visible from tablet portrait up. On phones (<768px) it remains a top accordion or bottom sheet.

Verify all primitives still look right at the in-between widths. The Card padding may need to reduce slightly at md to keep the right-pane content readable.

## Acceptance criteria
- [ ] Survey uses split-pane from 768px width up
- [ ] Respondent rail visible on iPad portrait
- [ ] Tap targets remain 44px minimum on tablet
- [ ] Card padding adjusts at md breakpoint if needed

## Files
- `src/pages/Survey.tsx` — change breakpoint
- `src/ui/SectionNav.tsx` (if 1.6 lands first) — same change

## Out of scope
- Landscape phone layout (rare for forms; treat as small tablet)
- Custom tablet keyboard handling
