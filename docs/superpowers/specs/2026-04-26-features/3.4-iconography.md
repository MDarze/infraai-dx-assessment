# 3.4 — Iconography pass

**Priority:** P2 · **Effort:** S · **Theme:** Branding

## Problem
The ThemeToggle uses unicode glyphs (`●` `○` `◐`) for system/light/dark. These look like placeholders and render inconsistently across fonts/OSes. Same risk anywhere we lean on glyphs (➕ ✕ ▶).

## Solution
Adopt a consistent SVG icon set. Two paths:
- **Lucide React** (~6KB tree-shaken per icon, MIT) — covers everything, instantly familiar.
- **Hand-rolled SVG icons** in `src/ui/icons/` — zero dependency, exact control, but slower to build out.

Recommend Lucide. Replace:
- ThemeToggle: `Sun` (light), `Moon` (dark), `MonitorSmartphone` or `Contrast` (system)
- Locale toggle: `Languages` icon next to the AR/EN segments
- Header buttons: `Settings`, `RotateCcw` for reset
- Future: any inline icons in Results / Survey

Use `stroke="currentColor"` so they auto-flip with theme.

## Acceptance criteria
- [ ] All current unicode glyphs replaced with SVG icons
- [ ] Bundle size impact <10KB gzipped
- [ ] Icons inherit text color from parent
- [ ] Icons sized consistently (16px or 20px depending on context)
- [ ] No icon left as a unicode glyph in production code

## Files
- Add `lucide-react` dep
- `src/ui/ThemeToggle.tsx`, `src/ui/AppHeader.tsx`, others — replace glyphs

## Out of scope
- Icon-only mode for header (already covered by mobile responsive)
- Animated icons (e.g., morphing sun-to-moon) — defer
