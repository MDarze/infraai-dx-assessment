# 3.3 — Custom radio + checkbox styling

**Priority:** P1 · **Effort:** S · **Theme:** Branding

## Problem
The QuestionRenderer in `pageHelpers.tsx` uses native browser radio/checkbox inputs. They render with OS-default chrome (blue/black squares, varying across Windows/macOS/iOS/Android), breaking the cohesion with the 2px brand geometry.

## Solution
Replace `<input type="radio">` and `<input type="checkbox">` with a `Radio` and `Checkbox` primitive that visually renders custom 2px-corner squares filled `bg-brand` when selected. Keep the native input as `sr-only` for accessibility. The `src/ui/Radio.tsx` primitive already exists per the spec but isn't used by QuestionRenderer; QuestionRenderer should adopt it.

A `Checkbox` primitive doesn't exist yet — create one, similar shape, with checkmark inside on selected state.

## Acceptance criteria
- [ ] All radio questions render with custom 2px brand squares
- [ ] All multi questions render with custom 2px brand checkboxes (with check glyph)
- [ ] Hover/focus visible (focus ring uses brand color)
- [ ] Keyboard nav still works (Tab, Space)
- [ ] Screen reader still announces selection state

## Files
- `src/ui/Radio.tsx` — already exists, may need tweaks
- `src/ui/Checkbox.tsx` (new)
- `src/pageHelpers.tsx` QuestionRenderer — use the primitives instead of native

## Out of scope
- Custom dropdown/select styling (done separately if needed)
- Toggle/switch primitive
