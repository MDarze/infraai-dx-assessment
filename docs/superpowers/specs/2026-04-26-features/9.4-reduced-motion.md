# 9.4 — Reduced-motion preference support

**Priority:** P3 · **Effort:** S · **Theme:** Accessibility

## Problem
Once we add animations (3.7 page transitions, 5.5 confetti, 1.1 progress bar transition), users with vestibular disorders or motion sensitivity will be discomforted. The OS `prefers-reduced-motion` media query exists for exactly this.

## Solution
Add a global CSS rule:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

Then individually verify the JS-driven animations (confetti, etc.) check the media query and skip themselves.

## Acceptance criteria
- [ ] Global CSS rule in styles.css
- [ ] All JS animations gated by `window.matchMedia('(prefers-reduced-motion: reduce)').matches`
- [ ] Verified by toggling OS preference
- [ ] No functionality broken when reduced

## Files
- `src/styles.css`
- Various components with JS animations

## Out of scope
- In-app toggle for reduced motion (defer to OS preference)
