# 10.4 — Versioning + changelog in footer

**Priority:** P3 · **Effort:** S · **Theme:** Operational

## Problem
The footer currently shows the current step name (e.g., "survey"), which is dev-tool noise. When supporting users on different deployed versions, you need to know which build they're on.

## Solution
Replace the step indicator in the footer with the build version (commit short hash + build timestamp). Inject at build time via Vite `define` config and a tiny shell snippet:

```js
// vite.config.ts
define: {
  __BUILD_HASH__: JSON.stringify(execSync('git rev-parse --short HEAD').toString().trim()),
  __BUILD_TIME__: JSON.stringify(new Date().toISOString()),
}
```

Footer renders `v{hash} · {date}`. Click to expand a small changelog modal with last 5 commits + dates.

## Acceptance criteria
- [ ] Version visible in footer
- [ ] Hash matches the deployed commit
- [ ] Optional: clickable to show recent changelog
- [ ] No PII (just hash and date)

## Files
- `vite.config.ts` — define block
- `src/App.tsx` Footer
- Optional: `src/changelog.json` generated at build time

## Out of scope
- Full changelog page
- Version skew warning (separate feature)
