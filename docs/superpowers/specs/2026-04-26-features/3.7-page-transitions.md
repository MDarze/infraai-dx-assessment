# 3.7 — Animated page transitions

**Priority:** P3 · **Effort:** S · **Theme:** Branding

## Problem
Step changes (Start → Survey → Results) are abrupt. The pages just swap. Tiny touch missing.

## Solution
Wrap step content in a transition wrapper that fades + slightly slides on step change. Use Framer Motion (heavier, ~25KB) or hand-roll with CSS transitions on a key prop (lighter, zero deps).

Recommend hand-rolled: opacity + translateY on enter, 200ms ease-out. Respect `prefers-reduced-motion` — disable entirely under that media query.

Keep duration fast (<250ms) so power users don't feel slowed down.

## Acceptance criteria
- [ ] Step changes fade-and-slide instead of pop
- [ ] Reduced-motion users see no animation
- [ ] No layout flicker during transition
- [ ] No measurable perf impact

## Files
- `src/App.tsx` — wrap step rendering
- `src/styles.css` — transition utilities

## Out of scope
- Per-question Survey transitions (separate, more invasive)
- Framer Motion adoption (overkill for this)
