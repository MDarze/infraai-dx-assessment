# 2.5 — Quick-win prioritization slider

**Priority:** P2 · **Effort:** S · **Theme:** Results value

## Problem
The recommendations are presented in a fixed analytical order. Different clients value different axes (a CFO cares about Finance recommendations; a project director about Process). One-size order isn't the right framing.

## Solution
Above the recommendations section, a small "Sort by:" segmented control with options: Default · Effort (low → high) · Impact (high → low) · Axis (group by axis label).

The data shape doesn't currently carry effort/impact as numeric fields — only as strings (`impactAr`/`impactEn`). Quick path: tag each quickWin in `analysis.ts` with `effort: 1|2|3` and `impact: 1|2|3`. Sort accordingly. Persist user's choice to localStorage so it sticks across sessions.

## Acceptance criteria
- [ ] Segmented control above quick wins on Results
- [ ] Order updates immediately
- [ ] Choice persisted across reloads
- [ ] Doesn't affect PDF export order (PDF stays default-sorted)

## Files
- `src/analysis.ts` — add effort/impact numeric tags to each quickWin entry
- `src/pages/Results.tsx` — segmented + sort logic
- `src/ui/Segmented.tsx` — already exists
- `src/i18n/{ar,en}.ts` — sort labels

## Out of scope
- Drag-to-reorder (segmented control is enough)
- Custom user weighting beyond the four presets
