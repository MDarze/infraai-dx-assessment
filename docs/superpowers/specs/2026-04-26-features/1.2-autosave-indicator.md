# 1.2 — Auto-save indicator

**Priority:** P1 · **Effort:** S · **Theme:** Survey completion

## Problem
The app already persists every answer to localStorage via `saveState()` in `App.tsx`, but there's no visible signal. Users on long surveys worry their work isn't being saved.

## Solution
A small "saved" pill that flashes when `saveState()` runs. Show in top-right of the Survey area, fade in for 200ms then fade out after 1.5s. Use `bg-brand/10 text-brand` for the success pill style already present in Results sync tag.

Implementation: add a `lastSavedAt` field to component state in `App.tsx` or a small hook; bump it on every `persist()` call; pass to Survey/Settings as a prop or expose via a global pubsub.

## Acceptance criteria
- [ ] Pill appears within 500ms of any answer change
- [ ] Pill auto-dismisses after ~1.5s
- [ ] Multiple rapid changes only show one pill (debounced)
- [ ] Pill labelled with `t("survey.autosave.saved")` (new key)
- [ ] Doesn't shift layout (absolute positioned)

## Files
- `src/App.tsx` — track lastSavedAt timestamp, pass down
- `src/pages/Survey.tsx` — render the pill
- `src/i18n/{ar,en}.ts` — add `survey.autosave.saved` ("تم الحفظ" / "Saved")

## Out of scope
- Sync-to-backend status (separate, on Results page)
- Save failure UI (localStorage failures are extremely rare)
