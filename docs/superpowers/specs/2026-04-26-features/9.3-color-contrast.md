# 9.3 — Color contrast verification (WCAG AA)

**Priority:** P1 · **Effort:** S · **Theme:** Accessibility

## Problem
The cream/charcoal palette is gentle but the muted tones (`ink-subtle` on `surface-alt`) likely fail WCAG AA contrast (4.5:1 for normal text). Failing contrast is hard for low-vision users and is a legal/procurement risk.

## Solution
Run the entire palette through a contrast checker. For every text/background pair used in real components:
- Compute ratio
- If <4.5:1 for body text or <3:1 for large text, adjust the token value

Common likely failures:
- `text-ink-subtle` (#8A8A85) on `bg-surface-alt` (#F3F1EA) → ~3.0:1, **fails**
- `text-ink-muted` (#5C5C58) on `bg-bg` (#FAFAF7) → ~6.0:1, OK
- Brand-on-brand-ink and danger-on-bg should both be checked too

Adjust the offending tokens. Document the chosen values and confirmed ratios.

## Acceptance criteria
- [ ] Every text/bg pair in production usage tested
- [ ] All pairs ≥4.5:1 (small text) or ≥3:1 (large text 18pt+)
- [ ] Both light and dark themes verified
- [ ] Changes applied to `src/styles.css` token definitions
- [ ] Visual diff: no surprising shifts in look

## Files
- `src/styles.css` — adjust `--c-ink-subtle` and any failing tokens
- Documentation of chosen ratios

## Out of scope
- High-contrast theme variant (separate)
- Color-blindness simulation
