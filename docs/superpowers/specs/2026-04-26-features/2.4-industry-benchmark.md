# 2.4 — Industry benchmark comparison

**Priority:** P2 · **Effort:** L · **Theme:** Results value

## Problem
A score in isolation is noise. "You scored 62/100" — is that good? Bad? Average? Without a reference frame the assessment lacks impact.

## Solution
For each axis, show the assessed company's score next to the median score of comparable peers (same `companySize`, same `industry`). Render as a small bar with two markers: client's score (filled brand) and peer median (outline).

Requires aggregation logic on the backend: for each `(axis, companySize, industry)` tuple, compute median across all completed assessments. Recompute nightly. Expose via `GET /api/benchmarks?companySize=...&industry=...`.

Don't show benchmarks until at least 10 peer assessments exist for the cohort (avoid noise). Show a "n=N peers" caption.

## Acceptance criteria
- [ ] Benchmark fetched on Results page load (only when sync.status === ok)
- [ ] Each axis stat shows client value + peer median
- [ ] If peer count < 10, hide benchmark for that axis with "Insufficient peer data"
- [ ] Benchmark cohort caption visible
- [ ] Privacy: no individual peer data exposed; only aggregated medians

## Files
- Backend: aggregation job + API endpoint
- `src/pages/Results.tsx` — benchmark renderer
- `src/api.ts` — fetchBenchmarks
- `src/ui/BenchmarkBar.tsx` (new)

## Out of scope
- Percentile bands (just median for v1)
- Filtering by city / region (cohort is size+industry only)
- Time-series benchmarks
