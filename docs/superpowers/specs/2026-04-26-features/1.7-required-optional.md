# 1.7 — Required vs optional question marking

**Priority:** P2 · **Effort:** S · **Theme:** Survey completion

## Problem
Currently no question is required; respondents skim and produce thin reports that compromise the analysis quality.

## Solution
Add `required?: boolean` to `QuestionDef`. Questions marked required show a small `*` after the text. Analyze button stays disabled until all required questions in the active respondent's set are answered (or explicitly skipped, depending on whether we want skipping to bypass required).

Decision needed: does skipping override required? Default: no, required means required. Skip is only available for non-required.

For the initial pass, mark a curated set of ~5–8 questions as required (chosen with the assessor based on what's analytically critical).

## Acceptance criteria
- [ ] `required: true` honored on questions
- [ ] Required marker visible on the question card
- [ ] Analyze button disabled with tooltip listing remaining required questions
- [ ] Skipped questions cannot be required (UI hides Skip on required)
- [ ] questions.json updated with required flags on chosen questions

## Files
- `src/types.ts` — extend QuestionDef
- `src/questions.json` — add `required: true` to chosen questions
- `src/pages/Survey.tsx` — render marker, hide Skip when required
- `src/App.tsx` — gate `finishSurvey` on requiredness
- `src/i18n/{ar,en}.ts` — `survey.required`, tooltip strings

## Out of scope
- Required only for some respondent roles (treat as global for now)
