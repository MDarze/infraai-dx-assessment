# 6.3 — Re-assess existing client (delta view)

**Priority:** P1 · **Effort:** M · **Theme:** Backoffice

## Problem
The strongest engagement story is "we measured you 6 months ago, here's what's changed." Currently each assessment is independent — no concept of "this is the same client revisited." Can't build a longitudinal pitch.

## Solution
On the assessment list (backoffice), select an existing assessment → "Re-assess" creates a new assessment record linked to the prior one (same `clientId` and same company meta auto-filled).

When the new assessment completes, the Results page can render a **delta view**:
- Score: 62 → 78 (+16)
- Per axis: shows arrows for axes that improved
- Recommendations: notes which prior recommendations were acted on (heuristic: improved scores in the axes they targeted)

Backend needs a `clientId` link between assessment records. Today `Company` is a separate entity per Prisma; if a Company already exists, the second assessment can attach to it.

## Acceptance criteria
- [ ] "Re-assess" button per client / per existing assessment
- [ ] New assessment pre-fills company meta
- [ ] Results page shows delta when a prior assessment exists
- [ ] Per-axis change visible
- [ ] Time-since-last-assessment shown

## Files
- Backend: link assessments to companies (already true?), endpoint to fetch prior
- Frontend: `src/api.ts` extends sync to optionally link to a prior assessmentId
- `src/pages/Results.tsx` — delta tile when prior available

## Out of scope
- Multi-snapshot trend chart (just two points: prior vs current for v1)
- Automated re-assessment reminders
