# 7.2 — AI-suggested follow-up questions

**Priority:** P2 · **Effort:** M · **Theme:** AI features

## Problem
A flat 25-question survey can't probe nuances. When a respondent answers "we use WhatsApp for updates" we want to ask "how many channels?" — but we can't ask everyone, since most won't have that pattern.

## Solution
After a respondent answers a question that triggers a flag (e.g., low score on Process axis), an LLM-generated follow-up appears inline as an optional probe. The probe is a free-text input; the response is stored as a note on the question.

Use Claude haiku for low-latency in-survey responses. Prompt engineered so the model only generates a follow-up for ~15% of answers (don't bloat the survey).

These follow-up notes feed into the AI executive summary (7.1) for richer narrative.

## Acceptance criteria
- [ ] Follow-up probes appear inline on flagged questions only
- [ ] User can dismiss without answering
- [ ] Responses persisted with the question answer
- [ ] No probe shown for >85% of questions (avoid survey bloat)
- [ ] Latency <1s for probe generation (or skip if slower)

## Files
- Backend: probe generation endpoint
- Frontend: Survey.tsx — render probe when flagged
- types.ts — extend Respondent to include probe responses

## Out of scope
- Multi-turn conversation per question (just one probe + response)
- Probe in PDF report (treat as internal data)
