# 8.1 — Webhook on assessment complete

**Priority:** P1 · **Effort:** S · **Theme:** Integrations

## Problem
On every successful assessment, you want to be notified — Slack, email, internal pipeline. Building each integration is overkill. Webhooks unlock all of them via Zapier / Make / n8n.

## Solution
Backend env var `COMPLETION_WEBHOOK_URL`. When `submit` succeeds (`/api/client/assessment/{id}/submit`), POST a JSON payload to that URL:

```json
{
  "event": "assessment.completed",
  "assessmentId": "...",
  "company": { "name": "...", "city": "...", "size": "...", "industry": "..." },
  "contact": { "name": "...", "email": "...", "phone": "..." },
  "score": 62,
  "axisScores": {...},
  "dna": {...},
  "createdAt": "2026-04-26T..."
}
```

Add `WEBHOOK_SIGNING_SECRET` for HMAC-SHA256 signature in `X-Signature` header so receivers can verify. Retry 3 times with backoff on non-2xx response.

## Acceptance criteria
- [ ] Webhook fires within 5s of assessment submit
- [ ] Signed with HMAC-SHA256 if secret configured
- [ ] Retry on failure
- [ ] Failure logged but doesn't break the user-facing submission
- [ ] Easily disabled (empty env var)

## Files
- Backend only
- Documentation in backend README

## Out of scope
- Multiple webhook URLs (one is enough; user can fan out via Zapier)
- Webhook delivery dashboard (use logs)
