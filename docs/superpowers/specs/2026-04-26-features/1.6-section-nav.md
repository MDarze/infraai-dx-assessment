# 1.6 — Question groups with section navigation

**Priority:** P2 · **Effort:** M · **Theme:** Survey completion

## Problem
The 25 questions span 8 axes (Process, DailyOps, Data, Finance, Governance, Decision, Automation, AIReadiness). Linear navigation forces respondents through every question even if they want to focus on, say, Finance only.

## Solution
On desktop (≥1024px), add a left rail showing the 8 axes as collapsible sections, each listing its questions with status (answered ✓ / current ● / skipped ◌ / pending ○). Click a question to jump to it. The rail stays in sync with the current question.

On mobile, the rail becomes a top accordion that collapses by default; tap to expand.

## Acceptance criteria
- [ ] Left rail (240px) on desktop, accordion on mobile
- [ ] Each axis labeled with `axisLabel(axis)` (already locale-aware)
- [ ] Per-question status icons reflect real state
- [ ] Click a question jumps to it; current question highlighted
- [ ] Survey URL doesn't change (no routing) but `idx` updates

## Files
- `src/pages/Survey.tsx` — add rail layout
- `src/ui/SectionNav.tsx` (new) — the rail component
- No new state needed; derive from existing answers + skipped set

## Out of scope
- Deep-linking to a specific question (no routing yet)
- Keyboard nav of the rail itself (just clicks for now)
