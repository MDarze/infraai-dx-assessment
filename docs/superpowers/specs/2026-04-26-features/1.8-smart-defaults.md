# 1.8 — Smart defaults from company size / industry

**Priority:** P3 · **Effort:** M · **Theme:** Survey completion

## Problem
Some answers are statistically tied to company size and industry. Asking every respondent every question feels redundant for predictable correlations and pads the survey.

## Solution
After the first ~10 assessments, build a tiny lookup table mapping `(companySize, industry) → most-common-answer per question`. When a respondent loads a new question, pre-fill the most-common answer as a "suggested" selection (visually distinct, e.g., dotted underline) that they can confirm with Enter or override by selecting differently.

Note: this is a dual-edged sword. Pre-filling can bias answers and flatten the data. Only enable for questions where the prefilled answer is correct >70% of the time.

## Acceptance criteria
- [ ] Suggestion only appears after 10+ assessments seed the table
- [ ] Suggested answer visually distinct from a confirmed answer
- [ ] User-selected answer overrides suggestion
- [ ] Track in analytics: how often suggestions are accepted vs overridden
- [ ] Only enabled per-question via a whitelist

## Files
- New: aggregation script in backend or a static JSON shipped with the app
- `src/pages/Survey.tsx` — render suggested state in QuestionRenderer
- Likely needs Sentry/Analytics first (10.1, 10.2) to measure bias risk

## Out of scope
- ML / fancy prediction; this is just frequency lookup
- Dynamic recompute on every new submission (recompute weekly)
