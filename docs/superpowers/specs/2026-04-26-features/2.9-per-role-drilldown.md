# 2.9 — Per-role drill-down on Results

**Priority:** P2 · **Effort:** S · **Theme:** Results value

## Problem
`analysis.ts` already computes `axisScoresByRole` (Manager / Engineer / Finance / Operations) but the data isn't exposed in the UI. It's a finding in itself when roles disagree on the same axis.

## Solution
Below the aggregate axis scores, add an expandable "View by role" section. When expanded, show a small table: rows = axes, columns = roles. Each cell shows that role's axis score; cells with high disagreement (delta > 1.5 vs aggregate) highlighted in `text-danger`.

Above the table, a one-line summary: "Engineer and Finance disagree most on Data (3.4 vs 1.8)" — auto-generated from the largest delta.

## Acceptance criteria
- [ ] Collapsed by default
- [ ] Expanded view shows axes × roles table
- [ ] High-disagreement cells visually distinct
- [ ] Summary line auto-generated
- [ ] Empty cells (no respondent for that role) show "—"

## Files
- `src/pages/Results.tsx` — new collapsible section
- `src/ui/Disclosure.tsx` (new) — generic expandable section primitive

## Out of scope
- Visualizing disagreement as a separate chart
- Per-question per-role breakdown (too granular for v1)
