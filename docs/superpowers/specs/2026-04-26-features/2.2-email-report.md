# 2.2 — Email report directly to client

**Priority:** P1 · **Effort:** M · **Theme:** Results value

## Problem
After completing an assessment the assessor has to: download PDF → switch to email → attach → write subject/body → send. That's friction at the moment of highest engagement. Often the client never receives the report.

## Solution
"Email report" button on the Results page footer alongside Export PDF. Clicking opens a small modal with:
- To: pre-filled with `state.meta.contactEmail` (editable)
- CC: pre-filled with assessor's email (editable)
- Subject: templated `t("results.email.subject", { client })`
- Body: templated, brief, with the score + a CTA to schedule follow-up

Backend endpoint `/api/report/email` does the send (Resend / Postmark / SES). PDF attached. Returns success/failure to the client; show inline confirmation.

## Acceptance criteria
- [ ] "Email report" button on Results, only visible when sync is OK and email is set
- [ ] Modal with editable To/CC/subject; defaults reasonable
- [ ] Send triggers backend email send with PDF attached
- [ ] Success/failure feedback in the modal
- [ ] Email body is bilingual (top: AR, bottom: EN, or split based on client preference field)

## Files
- Backend: new `/api/report/email` endpoint, email service integration
- `src/pages/Results.tsx` — Email button + modal
- `src/ui/Modal.tsx` (new) — generic modal primitive
- `src/i18n/{ar,en}.ts` — email subject/body keys

## Out of scope
- Email tracking (opens/clicks)
- Email template editor
- Sending from arbitrary user accounts (use service domain)
