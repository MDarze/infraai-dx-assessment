# 6.6 — Notes & action items per assessment

**Priority:** P3 · **Effort:** M · **Theme:** Backoffice

## Problem
After running an assessment in person, the assessor often has informal observations that don't fit the structured questions ("their CFO seemed skeptical of the AI angle"). These get lost in WhatsApp or email. Same for follow-up action items.

## Solution
Backoffice assessment detail page gets a Notes tab and an Actions tab:
- Notes: freeform text + optional file attachments (photos, voice memos)
- Actions: simple checklist with assignee + due date

Persisted server-side. Visible to assessor team only (not to client portal if 6.5 lands).

## Acceptance criteria
- [ ] Assessor can add notes after assessment is completed
- [ ] File attachments supported (limit 5MB per file, 5 files per assessment)
- [ ] Actions checkable; mark done with timestamp
- [ ] Audit trail (who added what when)

## Files
- Backend: notes + actions tables, file storage (S3/Cloudflare R2)
- Backoffice frontend: detail page tabs

## Out of scope
- Full-blown task manager (use a real one if needed)
- Notifications on action due (defer)
