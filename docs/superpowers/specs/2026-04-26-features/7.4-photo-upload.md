# 7.4 — Photo upload per question

**Priority:** P3 · **Effort:** M · **Theme:** AI features

## Problem
Operations questions ("show me your current dashboard", "what does your daily report look like?") get richer answers if respondents can attach a photo. Today there's no way to.

## Solution
Add an optional camera/file input to questions tagged `allowPhoto: true` in `questions.json`. On submit, photos uploaded to S3/R2 and attached to the assessment record.

Phase 2 (deferrable): when AI summary (7.1) generates, include attached photos in the model context (Claude vision). Photos enrich the narrative ("the screenshot shows manual Excel tracking — confirms the Data axis score").

## Acceptance criteria
- [ ] Photo upload available on flagged questions
- [ ] Mobile native camera capture (capture="environment" attribute)
- [ ] Image preview before save
- [ ] File size limit 5MB; auto-compressed client-side if larger
- [ ] Photos viewable in backoffice and PDF export

## Files
- Backend: file storage, presigned upload URLs
- Frontend: photo input component
- questions.json — `allowPhoto` flag on chosen questions

## Out of scope
- Video upload
- Auto-OCR / text extraction (defer to AI summary phase)
- Drawing on photos
