# 1.5 — Resume banner on app load

**Priority:** P1 · **Effort:** S · **Theme:** Survey completion

## Problem
When persisted state is found, `App.tsx` silently drops the user back into the Survey at question 0. They lose context: what client was this? Where was I? How do I start over?

## Solution
On app load, if persisted state exists, **don't** auto-navigate. Show a banner on the Start page:

> **Continue assessment for {clientName}** — last saved {relative time ago}
> [Continue] [Start fresh]

Continue → navigate to Survey at the last-edited question (track `lastQuestionIdx` in state).
Start fresh → call existing `clearState()` and reset.

If the persisted state has no clientName yet (just opened fresh), no banner — go to Start as today.

## Acceptance criteria
- [ ] First load with no persisted state: Start page, no banner
- [ ] Load with persisted state and clientName: Start page with banner offering Continue/Start fresh
- [ ] Continue resumes at the last visited question, not always question 0
- [ ] "Start fresh" clears localStorage and shows Start cleanly
- [ ] Banner uses brand styling and is dismissible

## Files
- `src/App.tsx` — change initial step logic; track lastQuestionIdx
- `src/pages/Start.tsx` — render banner conditionally above the title
- `src/storage.ts` — extend persisted shape with lastQuestionIdx
- `src/i18n/{ar,en}.ts` — keys for banner copy

## Out of scope
- Multiple in-progress assessments (still single-state for now)
- Cloud-stored draft (localStorage only)
