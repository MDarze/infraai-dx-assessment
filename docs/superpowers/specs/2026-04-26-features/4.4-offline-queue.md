# 4.4 — Offline-first verification with sync queue

**Priority:** P2 · **Effort:** M · **Theme:** Mobile

## Problem
The PWA already caches the app shell (vite-plugin-pwa) so it loads offline. But:
1. There's no explicit "offline" indicator — users don't know they're disconnected.
2. If `syncToBackend` is called offline, it just errors. No retry queue.

Field sites have spotty connectivity; this is a real risk.

## Solution
**Online indicator:** small dot in the AppHeader corner — green when navigator.onLine, gray when offline. Tooltip explains.

**Sync queue:** on save attempt with `navigator.onLine === false`, mark `sync.status = "queued"` and store a queue entry in localStorage. Listen for `online` event; on reconnect, replay queued submissions in order. Show a queued-count badge ("1 pending sync") in the header until cleared.

## Acceptance criteria
- [ ] Online/offline indicator in AppHeader
- [ ] Submissions queue when offline, auto-retry on reconnect
- [ ] Queue persists across app close/reopen
- [ ] Queued status visible on Results page
- [ ] No duplicate submissions if user retries manually after auto-retry succeeds

## Files
- `src/api.ts` — queue logic
- `src/storage.ts` — queue persistence
- `src/ui/AppHeader.tsx` — online indicator + queued badge
- `src/pages/Results.tsx` — render queued state

## Out of scope
- Conflict resolution if backend changes between attempts (last-write-wins)
- IndexedDB-backed queue (localStorage is enough for the data volume)
